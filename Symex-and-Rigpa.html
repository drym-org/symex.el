2.4 Symex and Rigpa ¶

Rigpa is a structured and flexible way to manage modal user interfaces like Symex and Evil. It allows you to visually define transitions between different modes via keybindings like Enter and Escape, making it convenient to use modal UIs (such as Symex and Evil) together. Note that the package, although fairly mature, is not yet available on Emacs package archives, and must be installed directly from the source repository using Straight or Elpaca. In addition to installing Rigpa, you’ll also need the symex-evil package mentioned in the previous section (currently, Rigpa does assume that you are an Evil user) as well as symex-rigpa which integrates Symex with Rigpa for smooth operation.

Please see the Rigpa documentation for instructions on installing Rigpa, and See Installing Symex for instructions on installing symex-evil and symex-rigpa.
